# Document Registry

A small dApp that proves a file has not been changed. Pick a file, the
browser computes its SHA-256 fingerprint, the fingerprint goes onto the
Polygon Amoy testnet. Anyone with the same file can recompute the
fingerprint later and verify it still matches what's on-chain. Files
themselves never leave your computer.

No wallet install required — the live site signs transactions for you using
a shared throwaway wallet stored as a Vercel env var.

---

## What changed across versions

**v1** ran on a local Hardhat node that wiped every restart, with an Express
backend in front. Impossible to demo from anywhere but your laptop.

**v2** is:

- **Persistent.** The contract lives on Polygon Amoy, a real public testnet.
  State survives forever, accessible from any browser.
- **Free.** Amoy MATIC from faucets; Vercel hobby plan; public RPC.
- **Visitor-friendly.** Frontend hashes locally; three tiny Vercel
  serverless functions (`/api/info`, `/api/register`, `/api/remove`) sign
  the on-chain transactions using a server-side throwaway wallet. The
  professor (or anyone) can click around without installing MetaMask.
- **Hash-only.** The app doesn't store the original file anywhere; just
  the fingerprint. To verify, the user re-supplies the file.

## Files

- `contracts/DocumentRegistry.sol` — per-wallet hash registry
- `scripts/deploy.js` — deploys the contract, writes `frontend/contract.js`
- `api/_contract.js`, `api/register.js`, `api/remove.js`, `api/info.js` —
  Vercel serverless functions that sign transactions with `PRIVATE_KEY`
- `frontend/index.html` + `frontend/app.js` — static UI (ethers via CDN)
- `frontend/contract.js` — auto-generated by deploy; address + ABI
- `test/DocumentRegistry.test.js` — Hardhat unit tests
- `DEPLOY.md` — step-by-step deploy guide (start here)

## Quick start

```bash
# Generate a throwaway wallet + fund it from the Amoy faucet, then:
cp .env.example .env       # paste the private key after PRIVATE_KEY=
npm install
npm run compile
npm run deploy:amoy

# Local dev including the /api endpoints:
npm install -g vercel
vercel dev
```

For the full walkthrough — including getting free MATIC, deploying to
Vercel, and the one Vercel env var you need to set — see
**[DEPLOY.md](./DEPLOY.md)**.

## Tests

```bash

```

The suite covers registration, per-wallet namespacing, overwrite prevention,
event emission, removal, and re-registration.

## How verification works

1. User picks a file in the browser.
2. The browser hashes it locally with SHA-256 (`crypto.subtle.digest`).
3. To register, the hash + file name + chosen ID are POSTed to `/api/register`.
   The serverless function signs and submits a transaction to the contract
   using the server's wallet.
4. To verify, the app reads the hash for that document ID from the contract
   over a public RPC, hashes the user's current file locally, and compares.
   Identical hashes mean identical bytes — even a single-bit change produces
   a totally different SHA-256.

The contract itself namespaces documents by uploader address, so although
this deployment uses a single shared signer, the same contract can be reused
by other wallets without collision.
