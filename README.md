# Document Registry

A small dApp that proves a file has not been changed. You pick a file, the
browser computes its SHA-256 fingerprint, and that fingerprint goes on the
Polygon Amoy testnet under your wallet. Anyone with the same file can
recompute the fingerprint later and verify it still matches what's on-chain.

Files themselves never leave your computer.

---

## What changed from v1

The original version ran on a local Hardhat node that wiped every restart, with
an Express backend in front of it. That made it impossible to demo from
anywhere but your laptop.

The new version is:

- **Persistent.** The contract lives on Polygon Amoy, a real public testnet.
  State survives forever, accessible from any browser.
- **Free.** Amoy MATIC is free from faucets; Vercel's hobby plan is free.
- **Serverless.** No backend. The browser hashes the file with Web Crypto API
  and signs transactions through MetaMask. The static site is hosted on Vercel.
- **Hash-only.** The app no longer stores the actual file; just its
  fingerprint. To verify, the user re-supplies the file.

- `frontend/index.html` — the static site

## Quick start

```bash
cp .env.example .env       # put your throwaway-wallet private key in here
npm install
npm run compile
npm run deploy:amoy        # writes frontend/contract.js
npm run serve              # opens http://localhost:3001
```

Then connect MetaMask, switch to Polygon Amoy when prompted, and start
registering files.

For full instructions (including how to get a free wallet, faucet MATIC, and
deploy to Vercel), see **[DEPLOY.md](./DEPLOY.md)**.

## Tests

```bash
npm test
```

The suite covers registration, per-wallet namespacing, overwrite prevention,
event emission, removal, and re-registration.

## How verification works

1. User chooses a file in the browser.
2. The browser hashes it locally with SHA-256 (`crypto.subtle.digest`).
3. To register, the hash + file name + chosen ID are written to the
   `DocumentRegistry` contract via a transaction signed by MetaMask.
4. To verify, the app reads the hash that was stored on-chain for that
   (wallet, ID) pair, hashes the user's current file, and compares. Identical
   hashes mean identical bytes — even a single-bit change produces a totally
   different SHA-256.

The contract namespaces documents by uploader address, so different wallets
can use the same `documentId` without collision.
